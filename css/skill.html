    # 基于伪元素的图片内容生成技术

    p52 4/1-2

    原理：FireFox在没有替换内容并且Chrome在此基础上有不为空的alt属性值，那么替换元素就会被当做内联标签

    想要该特性生效的技术点

    1. 不能有src属性（关键点所在）

    2. 不能使用content属性生成图片；（chrome）

    3. 需要拥有alt属性并有值； （chrome）

    4. Firefox下：before伪元素的content值会被无视，after无此问题。

    核心代码

    img:after: {
    /*生成的alt信息*/
    content: attr(alt);
    /*尺寸和定位*/
    position: absolute; bottom:0;
    width: 100%;
    background-color:rgba(0,0,0,0.5);
    transform: translateY(100%);
    /* 动效*/
    transition: transform .2s
}
img:hover::after{
    transform,: translateY(100%)
}

    # 通过hover改变图片效果

    <img src='laugh.png'>
<style>
    img:hover{
         content: url(laugh-tear.png)
    }
</style>

    该方法仅仅只是改变了视觉呈现，其src并没有改变
