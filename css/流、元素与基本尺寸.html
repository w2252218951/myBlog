    # 流、元素与基本尺寸

    # 1、块级元素:

    块级元素基本特征：在一个水平流上只能单独显示一个元素，多个块级元素则换行。（换行特性）

    清除浮动

    .clear:after{
    content: '';
    display: block;
    clear: both;
}

    list-item(附加盒子)生成了一个专门用来存放圆点、数字这些项目的'标记盒子'(mark box)。

    iE下不支持用伪元素创建这个盒子

    # width

    # 外部尺寸与流体特性

    1. 正常流宽度

    2. 格式化宽度： 定位是自动适应宽度

    # 内部尺寸与流体特性

    1. 包裹性：除了包裹性还有自适应，自适应指的是元素尺寸由内部元素决定；

    2. 首选最小宽度：

      规则：东亚文字最小为每个汉字的宽度

      西方文字最小宽度由特定的连续的英文字符单元决定

    3. 最大宽度：实际等同于包裹性，元素设置white-sapce:nowrap声明后的宽度

      如果内部没有块级元素或者块级元素没有设置宽度值，则最大宽度实际上是最大的内联盒子的宽度

    # width值作用的细节

    CSS盒模型从外到内

    1. margin:

    2. border

    3. padding

    4. content

    目前没有任何场景需要用到margin box

    因为margin相当于大气层，其背景永远是透明的。因此不能作为background-clip或background-origin的属性值

    margin一旦设定了具体的宽度和高度，其本身尺寸是不会因为margin值变化而变化的

    content box是围绕着width和height给定的矩形

    固定宽度值的缺点

    1. 流动性丢失：

      对于块状元素，设置width:auto，则元素会如水流般填充整个容器，一旦设置固定值，就会阻断这种流动性

    2. 与现实世界表现不一致

      width是直接作用在content box上的

    # CSS流体布局下的宽度分离原则

    所谓“宽度分离原则”，就是css中的width属性不与影响宽度的padding/border/margin属性共存

    也就是不能出现这种组合

    .box{
    width: 100px;
    border: 1px solid
}

    为什么要宽度分离？

    在前端领域，一提到分离，作用一定是便于维护。

    因为盒尺寸中的四个盒子都会影响到宽度，从而产生意想不到的布局

    # 4、改变widht/height作用细节的box-sizing

    box-sizing是css3的属性，但是IE8支持，不过要加-ms-前缀

    # 1、box-sizing（盒尺寸）的作用

    box-sizing属性的作用是改变width的作用细节

    .box1 { box-sizing: conent-box}  /* 默认值 */
.box2 { box-sizing: padding-box} /* Firefox曾经支持 */
.box3 { box-sizing: border-box}  /* 全线支持 */
.box4 { box-sizing: margin-box}  /* 从未支持 */

    将元素的box-sizing设置为 border-box，可以将设置的宽度直接作用在border-box上，从默认的content-box变为border-box,将content-box从宽度中释放，形成局部流动性

    # 2、为什么box-sizing不支持marin-box

    1、因为设置margin-box并没有实际价值，当我们限定宽度，再设置margin时，其宽度并没有发生变化。margin只有在设置为width:auto时才可以改变尺寸，但这是元素已经处于流动性了，根本不需要box-sizing

    2、margin所在的盒子区域永远是透明的，如果将其设置为一个“显示的盒子”，那么background-origin就很难规范

    3、离不开box-sizing属性的是

    普通文本框<input>和文本域<textarean>100%自适应父容器的宽度

    因为<textarea>为替换元素，替换元素的尺寸由内部元素决定

    box-sizing被发明出来的初衷就是解决替换元素宽度自适应问题

    box-sizing最佳使用技巧

    /*通过box-sizing: border-box  解决替换元素自适应宽度问题*/
input , textarea, img, video, object{
    box-sizing: border-box
}

    # 3、百分比高度值要起作用，其父级也要有有效高度值

    # 浏览器渲染基本原理

    先加载文档内容，加载头部的样式资源，然后按照自上而下，自内而外的顺序渲染DOM内容，基本上就是先渲染父内容，再渲染字内容。

    # 3.1、为何height：100%无效

    答：因为如果包含块的高度没有显示指定（高度由内容决定），并且该元素不是绝对定位，则计算值为auto

    包含块的宽度取决于该元素的宽度，就会产生未定义布局

    # 3.2、如何让元素支持height:100%效果

    1. 设置显示的高度值

    2. 使用绝对定位

      div{
    height: 100%;
    position: absolute
}

      此时的height会有计算值，即使祖先的height为auto

    绝对定位的百分比计算和非绝对定位元素百分比计算的区别

    • 绝对定位的宽高计算是相对于padding-box的，也就是会把padding-box计算在内

    • 非绝对定位则是相对于content-box计算的。

    # 3.3、超越！important ！超越最大！

    1、超越！important

    只要是一名前端开发就会知道，在css中！important的权重是最大的，但是有一个列外，当他的宽度设置为max-width时，会直接将!important的宽度进行覆盖

    <img src='' style='width:480px;!important;'>

    img{
     min-width: 250px   
}

    上述图片的宽度为250px

    2、超越最大

    超越最大指的是，min-width覆盖max-width,也就是当min-width和max-width同时存在的时候，以min-width为主

    # 3.4、任意高度展开收起动画

    当我们想对一个内容元素动态的（高度不固定）的元素设置展开动效时，我们可以在这个元素上添加max-height属性。我们只需要保证设定值比展开内容高度更大就行。

    因为max-height的值比height更大时，元素的高度就是height的高度

    .element {
    max-height: 0,
    overflow: hidden;
    transition: max-height .25s;
}
.element.active{
    max-height: 250px
}

    # 内联盒模型

    1. 内容区域：

      内容区域指一种围绕文字看不到的区域，我们可以把文本选中的背景色区域，当做内容区域。文本选中区本质上就等同于基本盒尺寸的content-box

    2. 内联盒子：

      “内联”盒子实际上指的是“外在盒子”，用来决定元素是内联还是块级，并且还分为“内联盒子”和“匿名内联盒子”

      内联盒子：外部包含内联标签<span>、<a>、<em>；

      匿名内联盒子：光秃秃的文字

    3. 行框盒子:

      每一行就是一个“行框盒子”，每个“行框盒子”又由一个个“内联盒子”组成

    4. 包含盒子，跟准确一点是包含块

      一个块级标签所在的就是一个包含盒子，一个包含盒子由多个“行框盒子”组成
